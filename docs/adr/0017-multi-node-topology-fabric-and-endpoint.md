## Context

The module (ADR-0016) must express multi-node clusters even while production stays single-node, and a persistent multi-node cluster is a **gated** future step (enter on real HA need, ‚â• 2 workers, benchmark pass, and non-shared-vCPU control-plane hardware ‚Äî #341). The long-term vision is **single-region, multi-provider**: control plane in one metro, workers on whichever bare-metal provider is best per workload. Candidates verified 2026-07-08: Cherry Servers and Latitude.sh (verified end-to-end: official Terraform providers plus productized iPXE‚ÜíImage Factory boot paths), Vultr (official provider and iPXE too, BIOS-only images), Hetzner (official provider for cloud; Robot dedicated is community-provider territory), servers.com (official provider, no productized iPXE).

Verified constraints that shape every choice below (fact-check record, 2026-07-08/09):

- **etcd latency envelope** (etcd Tuning/FAQ; OpenShift practices): heartbeat ‚âà RTT, election ‚â• 10√ó RTT, same values on all members; working budgets WAL-fsync p99 < 10 ms, backend-commit p99 < 25 ms, peer-RTT metric p99 < 50 ms (100 ms is the hard ceiling); every write costs ‚â• 1 RTT to quorum + fsync ‚Äî etcd latency *is* control-plane latency.
- **Provider private networks are intra-provider (mostly intra-facility)** ‚Äî Cherry VLANs, Latitude private networks, servers.com L2 segments, Hetzner vSwitch. A multi-provider control plane, even at 2 ms in one metro, must ride public-internet WireGuard. The fabric choice therefore decides whether a multi-provider CP is possible at all.
- **Tailscale is always userspace WireGuard** (wireguard-go; kernel dataplane was never implemented), and the Talos extension additionally defaults to netstack (`TS_USERSPACE=true`) ‚Äî the slowest path. TUN mode reaches ~7‚Äì13 Gbps with GSO/GRO on modern CPUs; etcd's small writes are latency/jitter-sensitive, not throughput-bound. Direct P2P is effectively assured between publicly-addressed bare metal once UDP 41641 is open; DERP relay fallback is disqualifying for etcd.
- **KubeSpan** is Talos-native kernel WireGuard full mesh (UDP 51820, MTU = underlay ‚àí 80, default 1420) ‚Äî but peer discovery depends on `discovery.talos.dev`, and self-hosting the discovery service **requires a commercial license**.
- **Cilium interactions**: VXLAN tunnel mode (ADR-0015 pin) works over WireGuard fabrics provided **eBPF host routing is disabled** (`bpf.hostLegacyRouting: true` ‚Äî the component that breaks TCP over WG interfaces per siderolabs/talos#11235; `bpf.masquerade` could then even stay on, though it stays off here per ADR-0015). MTU must be pinned explicitly (Cilium auto-detects from the 1500-MTU NIC, not the tunnel path) ‚Äî 1230 over Tailscale (1280 ‚àí 50), underlay ‚àí 130 over KubeSpan.
- **Talos mechanics**: `cluster.etcd.advertisedSubnets` takes effect only after a node reboot (siderolabs/talos#9605); etcd `extraArgs` behave the same (etcd restarts only on reboot) ‚Äî so subnet pinning must precede the first multi-node join. The shared VIP requires L2 adjacency and must never front the Talos API :50000 (recovery-path loss); kubeconfig carries a single server URL while `talosconfig` fans out across endpoints; KubePrism (localhost:7445) serves in-cluster API load-balancing; **CP‚Üíworker in-place demotion is unsupported** ‚Äî demotion means wipe and rejoin.
- **Remote workers are latency-tolerant by design**: kubelet lease renewal every 10 s against a 40‚Äì50 s grace period makes steady-state RTT a non-issue; the real failure mode is **eviction churn on flaky links** (`unreachable` taint ‚Üí pod eviction after `tolerationSeconds` = 300 s).
- Lessons standing from this repo: identifiers never in public logs as variables (#300); trigger identity never changes outside a planned reinstall window (#268); Tailscale ACLs gate reachability per tag (PR #299); hostname pinning is mandatory per node (#304).

## Considered Options

| Node fabric (first-class) | Works across providers w/o L2 | Dataplane | External dependency | Verdict |
|---|---|---|---|---|
| **Tailscale (opinionated default)** | Yes | Userspace wireguard-go (TUN mode required; netstack default must be disabled) | Tailscale coordination plane ‚Äî with **Headscale as an OSS self-host escape hatch** | **Selected** ‚Äî one fabric for every provider, existing integration and ACL investment, no L2 assumptions; viability on the CP tier to be *proven, not assumed* (decision 3) |
| KubeSpan | Yes | Kernel WireGuard | `discovery.talos.dev`; self-hosting requires a **commercial license** ‚Äî no OSS escape hatch | Second-class (flag-gated benchmark arm); never first-class on the licensing lock-in alone |
| Provider private network / vRack | **No** ‚Äî intra-provider only; contradicts the multi-provider vision as a default | Native kernel | None | Second-class; the documented CP-tier fallback, promotable to first-class only on user demand |
| Public-internet plaintext + firewall | Yes | Native | None | Rejected ‚Äî unencrypted etcd/kubelet traffic across providers is not a real option |

The support-tier policy (ADR-0016 decision 7) is the enforcement mechanism: exactly one fabric is Tier 1; the others are flag-gated Tier 2.

## Decision

**One opinionated, provider-agnostic fabric ‚Äî Tailscale ‚Äî with its known weaknesses engineered out as first-class requirements; alternatives kept as measured, flag-gated arms; every latency-sensitive claim settled by benchmark before this ADR is accepted.**

1. **First-class fabric: Tailscale, with mandatory mitigations** (module inputs, #324/#325):
   the extension runs in **TUN mode** (`TS_USERSPACE=false` ‚Äî the netstack default is the slowest path and never carries cluster traffic); `cluster.etcd.advertisedSubnets`/`listenSubnets` and `machine.kubelet.nodeIP.validSubnets` are pinned to the tailnet range (`100.64.0.0/10`) **before the first multi-node join** (changing them later costs a rolling CP reboot, talos#9605); Cilium MTU is **pinned to 1230** (never auto-detected); **eBPF host routing is disabled and `bpf.masquerade` pinned off** (Cilium must not auto-enable either over the WireGuard fabric ‚Äî this amends ADR-0015 decision 5's "leave `hostLegacyRouting` unset" stance for multi-node, recorded via #340); etcd `heartbeat-interval`/`election-timeout` are tuned to measured RTT (same values on all members); UDP **41641 ingress** is part of the node firewall matrix so direct paths are guaranteed (DERP fallback on any CP pair is a deployment-blocking finding, not a tuning input).
   **Boot sequencing is retry-converged, not ordered**: etcd and kubelet are pinned to a subnet that exists only once tailscaled is up, authenticated, and addressed ‚Äî they crashloop-retry until then, at genesis bootstrap and at every join. The sharp failure mode: a consumed-but-unpersisted single-use auth key leaves the node permanently without a tailnet identity ‚Äî etcd/kubelet never converge, and recovery is out-of-band (iKVM/config-drive rewrite, cf. the device-cleanup machinery #312). The runbook documents the convergence timeout and this failure signature.

2. **Second-class arms** (Tier 2, flag-gated): **KubeSpan** (adds the UDP 51820 rule-class; excluded from first-class because node-to-node connectivity must not depend on a service that cannot be self-hosted without a commercial license) and **provider private networks** (vRack/vSwitch-class; promotable to first-class on user demand ‚Äî they can never be the portable default since they end at the provider boundary).
   **Guidance while this ADR is Proposed**: fabric-agnostic module inputs (node_cidrs, pod/service CIDRs, cluster IDs, the firewall-matrix shape) land now in #324/#325; the fabric *selection* and its Tailscale-specific defaults ship un-defaulted/opt-in until #333 clears the gate, so acceptance sets the opinionated default rather than a later PR walking one back.

3. **Acceptance benchmark ‚Äî this ADR's gate** (#332/#333): Tailscale-TUN, KubeSpan, and provider-private-net arms run on identical ephemeral 3-node fixtures against: WAL-fsync p99 < 10 ms, backend-commit p99 < 25 ms, peer-RTT p99 < 50 ms, **zero leader changes under load**, `etcd_server_proposals_failed_total` = 0 (peer RTT measured as `etcd_network_peer_round_trip_time_seconds` p99), plus a tc-netem RTT/jitter degradation matrix and a **Cilium-VXLAN-over-WireGuard encapsulation arm** (MTU verification, iperf3 throughput, fragmentation counters). Netstack mode is measured once, for the record.
   **Fixture topology must cover the thesis, not just the cheap case**: a same-provider 3-VM fixture exercises near-LAN direct P2P and validates only the mechanics; before flip-to-Accepted the Tailscale arm must also run **cross-provider/cross-region with real NAT in front of at least one member**, and every run asserts **no CP pair is DERP-relayed** (a relayed pair fails the run). **Benchmark discipline**: a pinned load profile (kube-churn/write-rate generator) and soak duration for every p99; pass = **all criteria** on the deployment-shaped arm ‚Äî marginal results activate the fallback rather than a judgment call. These thresholds are *disqualifiers*, not targets: the deployment design band for a permanent CP tier remains < 10 ms metro RTT (decision 7).
   **Documented fallback**: if Tailscale-TUN fails the CP-tier criteria, the control plane moves to a same-provider private network (etcd pinned there via `advertisedSubnets`) while workers stay on Tailscale ‚Äî recorded here so the failure path is a configuration change, not a redesign. If the VXLAN arm fails over the fabric, ADR-0015's pin gets a formal supersede note (#340) rather than silent divergence.

4. **HA cluster endpoint**: a DNS name resolving to all control-plane Tailscale IPs is the `cluster_endpoint` (kubeconfig's single server URL). The name is served by a resolver **outside cluster state** ‚Äî the operator's external DNS zone or tailnet custom records (multi-A behavior to be verified during #333; if custom records cannot serve multiple A records, the external zone is the answer) ‚Äî never in-cluster DNS, which is circular exactly when recovery needs it. Role division: the multi-A name serves operators, CI, and joining workers; `extraHostEntries ‚Üí 127.0.0.1` (control-plane-role-only ‚Äî it is wrong on workers) short-circuits CP-local resolution; KubePrism:7445 load-balances in-cluster clients. `talosconfig` lists every CP endpoint (native fan-out). The endpoint name plus every CP's tailnet FQDN and IP go into certSANs ‚Äî and the SAN set is re-managed (config re-apply per CP) on every node add/replace, including the decision-6 migration. A cloud LB is rejected (public 6443 violates ADR-0005's zero-trust posture). The Talos **VIP** is only a conditional refinement inside a single-provider shared-L2 branch, and **never** fronts the Talos API :50000. Known accepted limitation: kubectl does not fail over gracefully across A records ‚Äî a dead CP causes operator-side retries, not an outage.

5. **Worker join path** (resolved by the fabric choice): nodes receive their tailnet identity at provision time ‚Äî per-node single-use auth key delivered through the **adapter's** config-injection path (this extends ADR-0016's adapter contract with a per-node-identity element alongside node IPs/provisioned handle/trigger), while TUN mode, subnet pins, MTU, etcd tuning, and the firewall matrix live in **talos-core**. A worker's *first* contact with `cluster_endpoint` therefore already rides the tailnet ‚Äî no public 6443 exposure, no bootstrap exception in the firewall ‚Äî subject to the same retry-convergence and key-failure mode as decision 1. Cross-provider workers are exactly as reachable as same-provider ones. The tailnet ACL must grant node‚Üînode for the mesh matrix ports (ACL-as-code, #336).

6. **Migration to persistent multi-node** (executed under #341, gated) ‚Äî honestly characterized: this is a **new-CP build plus restore**, not an in-place scale-up, because demotion is unsupported. Sequence: stand up the new control plane from the module and verify etcd health there first ‚Äî **reusing the existing machine secrets** (PKI/CA continuity keeps kubeconfig/talosconfig trust, GitOps credentials, and CI secrets stable; accepted implication: two clusters transiently share PKI during the window); redeploy workloads via GitOps (ADR-0018's machinery); take a verified off-box ZFS backup; wipe the current server, rejoin it as a **worker** with a **fresh Tailscale hostname** (stale-device suffix trap, #312); then **restore the data pool** (`zfs send` back, per ADR-0018 decision 5(c)) ‚Äî stateful workloads are down for that restore window, bounded by ADR-0018's node-loss RTO tier. `talosctl bootstrap` never runs a second time. The post-gate control-plane topology is **3 nodes, one metro, < 10 ms RTT, non-shared-vCPU hardware**, default etcd timeouts.

7. **Multi-cluster threshold rule ‚Äî a hard rule, not a norm**: if per-region HA or region isolation is ever required, the answer is **independent clusters** joined by Cilium ClusterMesh (non-overlapping pod CIDRs and unique 1‚Äì255 cluster IDs are reserved as module inputs now, #324; cross-cluster WireGuard is all-or-nothing and needs UDP 51871) or a federation layer. **Measured CP peer-RTT p99 > 30 ms, or failure of any decision-3 etcd criterion on the candidate fabric, means independent clusters ‚Äî no exceptions**; the #333 benchmark harness is re-run as a mandatory pre-flight before any CP member is added on a new fabric, region, or provider (the #342 RTT/egress matrix feeds the candidate data). RTT design bands: < 10 ms comfortable; 10‚Äì30 ms workable with tuned heartbeat/election and slower writes; above the line, a second cluster.

8. **Remote-worker semantics**: adapters label nodes with provider/site topology (`topology.kubernetes.io/zone`-style, #328); guidance ships for `topologySpreadConstraints`, topology-aware routing, and `tolerationSeconds` tuning ‚Äî because the failure mode of far workers is eviction churn on flaky links, not RTT. Cross-provider east-west traffic is metered egress (cost model, #344).

9. **Firewall/mesh matrix as module API** (#325): role-aware rule sets parameterized by `node_cidrs` ‚Äî etcd 2379-2380 (CP-only), VXLAN 8472, kubelet 10250, apid/trustd 50000/50001, Hubble 4244/4245, Tailscale 41641 ingress (first-class), KubeSpan 51820 (flag-gated). Talos NetworkRuleConfig is the portable enforcement layer; per-provider security groups mirror it as defense-in-depth.

10. **Supersessions on acceptance** (executed via #340): ADR-0005 receives a scope-extension note (Tailscale grows from operator access to node fabric); ADR-0015 decision 5 receives the host-routing amendment from decision 1 (and a VXLAN supersede note only if the benchmark forces it); ADR-0012's remaining single-node half is superseded only when #341 actually ships.

## Consequences

### Benchmark record (#333)

*Placeholder ‚Äî filled by the flip PR with, per arm (Tailscale-TUN, netstack reference, KubeSpan, private net; same-provider and cross-provider fixtures): WAL-fsync p99, backend-commit p99, peer-RTT p99, leader changes under load, `etcd_server_proposals_failed_total`, DERP-relay assertion result, and the VXLAN arm's iperf3/MTU/fragmentation numbers ‚Äî plus the verdict and whether the decision-3 fallback activates.*

**Positive:**

- One fabric configuration works identically on every current and future provider ‚Äî no L2 assumptions, no per-provider network modules ‚Äî which is what makes the adapter model (ADR-0016) portable in practice.
- The coordination-plane dependency has an OSS self-host escape hatch (Headscale), unlike the rejected alternative's discovery service; the existing tailnet ACL investment and operational familiarity carry over.
- The endpoint, join-path, and migration designs contain no public API exposure at any step, preserving ADR-0005's posture through the topology change.
- The stretched-etcd trap is closed in writing ‚Äî with a hard threshold and a mandatory pre-flight ‚Äî before anyone is tempted.

**Negative / accepted risks:**

- **Userspace dataplane on the control-plane path** (pending benchmark): per-packet userspace processing adds jitter under CPU contention ‚Äî exactly what etcd heartbeats dislike. This is the risk the benchmark exists to quantify; the fallback in decision 3 is the pre-agreed exit.
- **Two distinct Tailscale-side dependencies**: (a) the **coordination plane** ‚Äî established WireGuard sessions survive an outage, but key expiry, new joins, and ACL changes do not; node key expiry is disabled for cluster nodes, and a coordination outage during a node join is an accepted (rare) failure mode, with Headscale as the documented exit. (b) **DERP relays during the join/handshake window** ‚Äî every new pairing transits DERP until direct P2P is negotiated, so joins and migrations depend on DERP reachability at boot even though steady-state relaying is disqualifying; publicly-addressed bare metal with 41641 open leaves DERP within seconds, and self-hosted DERP is the escape hatch if that tier ever matters.
- **The tailnet ACL is a single-config blast radius for the whole node mesh** (distinct from any Tailscale outage): one bad ACL-as-code push can drop etcd/VXLAN/kubelet across all N nodes at once ‚Äî and can cut the operator's own tailnet reach with it (the reachability-gating lesson of PR #299). Mitigations: ACL changes go through CI validation/preview and staged application (#336), and the out-of-band recovery path (iKVM/console, ADR-0005's emergency access) is documented for a self-inflicted mesh partition.
- **tailscaled version is coupled to the Talos schematic**: the fabric's transport cannot be patched independently of a Talos image change ‚Äî security patches ride the upgrade lifecycle, and rolling upgrades leave transient version skew across the mesh. Mitigation: the extension pin is managed like every other image pin, upgraded via the existing upgrade-mode machinery.
- **MTU tax**: 1230-byte pod MTU costs measurable throughput on cross-node traffic versus native 1450+; accepted for the portability, quantified by the benchmark's iperf arm.
- **Per-node tailnet identity multiplies key/ACL surface**: per-node auth keys, device cleanup at N nodes (#312), and ACL-as-code (#336) are prerequisites, not niceties.
- Benchmark-gated acceptance means fabric-specific defaults stay un-shipped until #333 lands (decision 2's guidance) ‚Äî deliberate sequencing, not indecision.
